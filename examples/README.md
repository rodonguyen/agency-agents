# Ví dụ

Thư mục này chứa các ví dụ cho thấy nhiều agent có thể phối hợp để xử lý nhiệm vụ thực tế.

## Vì sao có thư mục này?

Repo `agency-agents` định nghĩa nhiều chuyên gia AI trên các mảng engineering, design, marketing, product, support, spatial computing, project management và nhiều lĩnh vực khác. Nhưng chỉ đọc định nghĩa agent chưa đủ để thấy điều gì xảy ra khi **triển khai nhiều agent cùng lúc** cho một nhiệm vụ.

Các ví dụ trả lời câu hỏi: *“Khi cả agency cùng cộng tác thì kết quả trông như thế nào?”*

## Nội dung

### [nexus-spatial-discovery.md](./nexus-spatial-discovery.md)

**Là gì:** Một bài product discovery hoàn chỉnh, trong đó 8 agent chạy song song để đánh giá cơ hội phần mềm và tạo kế hoạch thống nhất.

**Bối cảnh:** Web research phát hiện cơ hội ở giao điểm giữa AI agent orchestration và spatial computing. Sau đó nhiều agent được triển khai đồng thời để tạo ra:

- Xác thực thị trường và phân tích cạnh tranh.
- Kiến trúc kỹ thuật, gồm thiết kế hệ thống 8 service và SQL schema đầy đủ.
- Chiến lược thương hiệu và nhận diện hình ảnh.
- Kế hoạch go-to-market và tăng trưởng.
- Blueprint vận hành customer support.
- Kế hoạch UX research với persona và journey map.
- Kế hoạch triển khai 35 tuần với 65 sprint ticket.
- Đặc tả kiến trúc spatial interface.

**Agent đã dùng:**

| Agent | Vai trò |
|-------|---------|
| Product Trend Researcher | Xác thực thị trường, bối cảnh cạnh tranh |
| Backend Architect | Kiến trúc hệ thống, data model, API design |
| Brand Guardian | Positioning, visual identity, naming |
| Growth Hacker | GTM strategy, pricing, launch plan |
| Support Responder | Support tier, onboarding, community |
| UX Researcher | Persona, journey map, nguyên tắc thiết kế |
| Project Shepherd | Phase plan, sprint, risk register |
| XR Interface Architect | Đặc tả spatial UI |

**Điểm rút ra:** 8 agent có thể chạy song song và tạo kế hoạch nhất quán, có tham chiếu chéo, mà không cần quá nhiều chi phí điều phối thủ công.

## Thêm ví dụ mới

Nếu bạn chạy một bài multi-agent thú vị, hãy cân nhắc thêm vào thư mục này. Ví dụ tốt nên thể hiện:

- Nhiều agent cùng cộng tác trên một mục tiêu chung.
- Bề rộng năng lực của agency.
- Tính ứng dụng thực tế của định nghĩa agent.
- Đầu ra đủ rõ để người khác học hoặc tái sử dụng.

Khi thêm ví dụ, giữ nguyên command, path, link và artifact kỹ thuật nếu có.
